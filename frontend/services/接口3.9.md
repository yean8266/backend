### **A:获取全局初始化数据 (Init Data)**

请求方式：GET /api/papers?sort={排序方式}\&page={页码}\&search={搜索词}

注意这里的sort page search 之后会需要从sql里把对应的文件给我。

比如我请求的是 最热 第二页 关键词没有，就要对应的文章，我展示在前端。

若前端未传 page，请默认视为 page=1

若前端未传 sort，请默认视为 sort='hot'

带 Token 返回个人票数，不带 Token 仅返回文章列表

      "date": "2026-03-01T12:00:00Z"

    }

  ]

}



**算法思路：**

1.跨天重置检测（关键算法）：如果解析到 Token（用户已登录），对比数据库中该用户的 last\_active\_date。如果该日期早于今天（北京时间 00:00），则触发重置算法：daily\_votes\_left = 5, has\_shared\_today = false, 更新 last\_active\_date 为今天。

2.聚合用户投票数据：如果用户已登录，去 vote\_logs 表查出该用户对这些文章分别投了多少票，并映射到文章对象的 userVotes 字段上。

3.构建查询语句 (SQL Builder)：

WHERE 条件：

如果前端传了 search 参数，追加条件：WHERE title LIKE '%搜索词%' OR author LIKE '%搜索词%' OR id LIKE '%搜索词%'。

若未审核通过的文章不可见，还要加上 status = 'approved'。

ORDER BY 排序：

若 sort === 'hot'，则 ORDER BY total\_votes DESC, created\_at DESC。

若 sort === 'new'，则 ORDER BY created\_at DESC。

分页 (LIMIT \& OFFSET)：

设定每页 20 条（Page Size = 20）。计算偏移量：LIMIT 20 OFFSET (page - 1) \* 20。



### **B:投它一票 (Submit Vote)**

请求方式：POST /api/vote

请求体：{ "paperId": "BNL-2603-01" }

期望返回：{ "success": true }



**算法思路（需防并发刷票）：**

开启数据库事务 (Transaction)。

加行锁（SELECT ... FOR UPDATE）锁住该 user 的记录。

校验 user.daily\_votes\_left > 0。若不足，回滚事务并返回 403 错误。

扣减票数：UPDATE users SET daily\_votes\_left = daily\_votes\_left - 1。

增加总票：UPDATE papers SET total\_votes = total\_votes + 1。

记录明细：在 vote\_logs 中 upsert（若无则插入记录 vote\_count = 1，若有则 vote\_count = vote\_count + 1）。

提交事务 (Commit)。



#### **接口 C：撤回一票 (Unvote)**

请求方式：POST /api/unvote

请求体：{ "paperId": "BNL-2603-01" }

期望返回：{ "success": true }

**算法思路（需防并发）：**

开启数据库事务。

加行锁查 vote\_logs，校验该 user 对该 paper 的 vote\_count > 0。若等于0或没记录，说明他在非正常请求，回滚事务并返回 400。

归还票数：UPDATE users SET daily\_votes\_left = daily\_votes\_left + 1。

扣减总票：UPDATE papers SET total\_votes = total\_votes - 1。

修改明细：UPDATE vote\_logs SET vote\_count = vote\_count - 1。

提交事务。

#### **接口 D：上报分享成功 (Report Share)**

请求方式：POST /api/share

请求体：空 (依赖 Token 识别用户)

期望返回：{ "success": true }

**算法思路：**

开启事务。

锁住 user 记录，检查 user.has\_shared\_today == false。如果已经是 true，说明今天领过了，回滚并返回 400。

奖励发放：UPDATE users SET daily\_votes\_left = daily\_votes\_left + 1, has\_shared\_today = true。

提交事务
